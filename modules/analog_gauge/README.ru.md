# Аналоговый датчик (canvas) — виджет дашборда Zabbix

[English](README.md) | **Русский** | [Srpski](README.sr.md) | [Polski](README.pl.md) | [Latviešu](README.lv.md)

Мульти-виджет **«сетка аналоговых манометров»**. Показывает значения нескольких айтемов
(выбранных паттерном имени и отфильтрованных по тегам) круглыми стрелочными
циферблатами, разложенными по сетке. Каждый циферблат плавно доводит стрелку до
текущего значения айтема; если данных нет — стрелка крутит медленную демо-развёртку.
Всё рисуется процедурно на `<canvas>` (без картинок-ассетов).

Это «сеточный» родственник виджета [Термометр](../thermometer): та же модель данных
(паттерн + теги + макросы по хосту + пороги), но раскладка **сеткой, а не горизонтальной
каруселью**, и с четырьмя оформлениями на выбор.

![Стили аналогового датчика](docs/styles.png)

- **id:** `analog_gauge` · **namespace:** `AnalogGauge` · **type:** `widget`
- Класс отрисовки: `WidgetAnalogGauge` (наследует общую базу `CWidgetGaugeBase` →
  `CWidgetCanvasBase`).

## Возможности

- **Несколько айтемов с нескольких хостов** — выбор **паттерном имени** (маски `*`) и
  фильтр по **тегам** (нативная модель *SVG graph*; `inheritedTags` учитывает и теги
  хоста). Показываются **только числовые айтемы** (float / unsigned); текстовые, лог и
  символьные отсекаются.
- **Раскладка сеткой**: каждый подходящий айтем — свой циферблат, разложены сеткой.
  Число колонок подбирается автоматически, чтобы циферблаты были как можно более
  квадратными (и крупными), либо фиксируется полем **Колонки сетки**.
- **Минимальный размер + прокрутка**: если задан **Мин. размер циферблата**, циферблаты
  не мельчают ниже него — то, что не влезло, достаётся **перетаскиванием мышью** по
  **обеим осям** (появляются тонкие индикаторы-скроллбары). При мин. размере `0` сетка
  всегда подгоняется под виджет (без прокрутки). Автоскролла нет — панорама только вручную.
- **Три стиля** с общей геометрией дуги 270°:
  - **Retro** — винтажный латунный безель, кремовый циферблат, засечные цифры, классическая
    чёрная стрелка.
  - **Cyberpunk** — тёмная шайба, неоновая дуга-прогресс и светящаяся стрелка,
    моноширинный readout.
  - **Industrial** — тяжёлый стальной безель с болтами, матовый графит, чугунная стрелка,
    жёлто-чёрная опасная зона / пороги у верха.
- **Пороги**: значения порогов задают **только цветные зоны на дуге**; цифровое значение и
  дуга-прогресс сохраняют фиксированный цвет стиля независимо от значения. Зоны можно
  выключить.
- **Пользовательские макросы**: **Min**, **Max** и значения порогов могут быть макросами
  (напр. `{$PRESSURE.MAX}`, `{$WARN}`). Раскрываются **по каждому айтему, в контексте его
  хоста** — один и тот же макрос на разных хостах может дать разные числа, поэтому у
  каждого циферблата свои шкала и уровни порогов.
- **Диапазон**: **Fixed** (из Min / Max) или **Auto** — единая общая шкала по объединённой
  истории за последний час **всех** айтемов (с запасом ±5 %).
- **Цифровое значение** (опционально) в центре / врезке каждого циферблата, с заданным
  числом знаков и единицами айтема (или переопределением).
- **Дрожание стрелки** (опционально): стрелка/указатель слегка дрожит, имитируя работающий
  прибор — глазу легче поймать движение, чем статичное положение. **Дрожит только стрелка;
  цифровое значение — никогда.**
- **Имя айтема** под каждым циферблатом (обрезается по ширине); при наведении — подсказка
  с полным именем и хостом.
- **Плавная анимация стрелки** у каждого айтема (значение доводится к цели на каждом
  обновлении); при отсутствии данных крутится демо-развёртка. **Учитывает тему** (лица
  Retro / Cyberpunk / Industrial несут свой фон; подписи следуют светлой / тёмной теме
  дашборда).

## Настройки

| Поле | Описание |
|------|----------|
| **Группы узлов** | *(только глобальный дашборд)* ограничить поиск хостов этими группами. |
| **Узлы** | *(только глобальный дашборд)* паттерн(ы) имени хоста, маски `*`. |
| **Паттерны айтемов** \* | паттерн(ы) **имени** айтема, маски `*`. Одна `*` = все числовые айтемы найденных хостов. |
| **Теги айтемов** | фильтр по тегам, вычисление **And/Or** или **Or**. |
| **Переопределить узел** | привязать все айтемы к одному хосту (напр. на шаблонном дашборде). |
| **Стиль** | Retro / Cyberpunk / Industrial. |
| **Диапазон** | **Fixed** (Min / Max) или **Auto** (общий, история за час ±5 %). |
| **Min**, **Max** | границы шкалы. Числа **или** макросы (`{$LOW}`, `{$PRESSURE.MAX}` …). |
| **Единицы (override)** | заменить единицы айтема на циферблате. |
| **Знаков** | знаков после запятой в цифровом значении (0–10). |
| **Колонки сетки** | число колонок; **0 = авто** (подгон под виджет). |
| **Мин. размер циферблата, px** | минимальный размер для авто-раскладки; **0 = подгон под виджет** (без прокрутки). При > 0 переполнение достаётся перетаскиванием. |
| **Показывать цифровое значение** | показать / скрыть числовой readout на циферблате. |
| **Дрожание стрелки** | стрелка слегка дрожит (имитация работающего прибора). |
| **Пороги** | цветные уровни; значения могут быть макросами. |
| **Показывать зоны порогов на дуге** | закрасить цветные зоны на дуге циферблата. |

## Поток данных

1. Контроллер раскрывает **паттерны имён** айтемов по найденным хостам (или
   шаблонному / override-хосту), с фильтром по тегам — модель как у SVG graph. Остаются
   только числовые айтемы (`value_type` по float / unsigned).
2. Для каждого айтема возвращается **последнее значение**, а в режиме **Auto** — общий
   min/max по объединённой истории за час всех айтемов (±5 %).
3. **Min / Max и каждая строка порога раскрываются по каждому айтему, в контексте его
   хоста** (макросы на разных хостах разные) и парсятся в числа через `CNumberParser`.
   Поэтому каждый айтем несёт свои `min`, `max` и отсортированные `thresholds`.
4. JS рисует сетку; каждый циферблат анимирует стрелку к своему значению по своей шкале и
   своим пороговым зонам.

## Замечания / реализация

- Общие базы `class.widget.base.js` и `class.gauge.base.js` — **assign-once глобалы**
  (`window.X = window.X || class …`): идентичные копии лежат в каждом canvas/gauge-модуле,
  класс определяет тот, кто загрузился первым. Нельзя переименовывать члены, уже занятые
  `CWidget` (`_body`, `_fields`, …).
- Пороги-**макросы** сохраняются строками через поле
  `Modules\AnalogGauge\Includes\CWidgetFieldGaugeThresholds` (штатное поле порогов
  выбрасывает нечисловые строки); контроллер раскрывает и сортирует их по хосту.
- Прокрутка — только ручным перетаскиванием мыши, с гейтом по `isEditMode()`, чтобы не
  конфликтовать с перетаскиванием самого виджета в режиме редактирования.
- Всегда линтить JS перед загрузкой (`node --check assets/js/*.js`): синтаксическая ошибка
  в ассете модуля роняет базовый класс на **всех** дашбордах, т.к. ассеты грузятся на всех
  страницах.

## Демо

Дашборд **«Analog gauge (grid) demo»** (id `709` на учебном инстансе) — по странице на
стиль на хостах `Demo sensors` с per-host макросами (`{$TEMP.MIN}`/`{$TEMP.MAX}`) и
порогами `{$TEMP.WARN}`/`{$TEMP.CRIT}`, плюс страница **«Grid + scroll»** (узкий виджет,
мин. размер 180 px) с демонстрацией прокрутки мышью. Дрожание стрелки в демо включено.
